# 道法术模型

## 核心框架

将任何信息/知识分为三层：

- **道**（中心思想）：核心论点、本质规律、你要表达的一句话结论
- **法**（方法论/论点）：支撑"道"的论证角度、分论点、方法体系
- **术**（案例/论据）：具体事例、数据、操作步骤

## 使用流程

### 1. 输入分析

收到信息后，先识别三层结构：
```
这段内容的核心观点是什么？→ 道
它从哪几个角度论证？→ 法
用了什么具体案例/数据？→ 术
```

### 2. 输出组织

任何表达（写作/汇报/prompt）都按 道→法→术 展开：
```
先说结论（道）
再列理由（法）
最后举例（术）
```

### 3. AI提示词优化

低效 prompt：`帮我总结这篇文章`
高效 prompt：
```
请用"道法术"框架分析这篇文章：
- 道：核心观点是什么？
- 法：从哪几个角度论证？
- 术：用了哪些案例/数据？
```

## 应用场景速查

| 场景 | 道 | 法 | 术 |
|------|------|------|------|
| 写文章 | 核心论点 | 分论点 | 案例数据 |
| 会议纪要 | 核心议题 | 讨论要点 | 关键发言 |
| 学习笔记 | 知识本质 | 方法论 | 具体例子 |
| 工作汇报 | 结论成果 | 推进逻辑 | 关键节点 |
| Prompt | 任务目标 | 约束条件 | 具体要求 |

## 概念思维三级（进阶）

信息还可以按抽象程度分级：
- **经验知识**（1级）：具象案例，感性认识
- **理性知识**（2级）：方法论，可迁移的规律
- **绝对知识**（3级）：抽象公理，放之四海皆准

笔记时有意识区分这三级，避免"记了一堆案例但抓不住规律"。
