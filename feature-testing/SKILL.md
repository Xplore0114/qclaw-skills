# 新功能上线规则

## 流程（必须遵守）
1. **先测试** — 每个新功能上线前，必须先在本地/测试环境验证
2. **记录结果** — 测试通过后，记录测试内容和结果
3. **小范围试跑** — 先运行一次确认效果
4. **正式上线** — 确认无误后才添加到生产环境
5. **更新文档** — 更新 MEMORY.md 和相关 skill

## 测试清单
- [ ] 功能是否按预期工作
- [ ] token 消耗是否在合理范围
- [ ] 是否影响其他功能
- [ ] 是否需要额外配置
- [ ] 错误处理是否完善

## 回滚方案
- 有问题时直接删除 cron job 或禁用 skill
- 保留测试记录，方便排查
